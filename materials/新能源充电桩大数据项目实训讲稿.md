## 第5课时：SSH免密登录与Hadoop集群启动

### 5.1 课程导入（5分钟）

Hadoop集群中，各节点之间需要通信。伪分布式模式下，NameNode需要免密登录本机的DataNode。所以我们先配置SSH免密登录。

### 5.2 SSH免密登录讲解（10分钟）

SSH免密登录的原理是**公钥认证**：

1. 在本机生成一对密钥（公钥+私钥）
2. 将公钥追加到`authorized_keys`文件中
3. SSH连接时，系统用公钥验证私钥，匹配则免密登录

### 5.3 实操：配置免密登录与启动Hadoop（20分钟）

```bash
# 1. 生成RSA密钥对（一路回车）
ssh-keygen -t rsa -P "" -f ~/.ssh/id_rsa

# 2. 将公钥追加到授权文件
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# 3. 测试免密登录
ssh localhost
# 第一次连接输入yes，应该不需要密码就能登录
exit
```

现在配置Hadoop的伪分布式模式。

```bash
# 4. 配置hadoop-env.sh
echo 'export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64' | sudo tee -a /opt/hadoop/etc/hadoop/hadoop-env.sh

# 5. 创建HDFS数据目录
mkdir -p ~/hadoopdata/hdfs/namenode
mkdir -p ~/hadoopdata/hdfs/datanode
```

配置`core-site.xml`：

```bash
cat > /opt/hadoop/etc/hadoop/core-site.xml <<'EOF'
<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://localhost:9000</value>
  </property>
</configuration>
EOF
```

配置`hdfs-site.xml`：

```bash
USER_HOME=$HOME
cat > /opt/hadoop/etc/hadoop/hdfs-site.xml <<EOF
<configuration>
  <property>
    <name>dfs.replication</name>
    <value>1</value>
  </property>
  <property>
    <name>dfs.namenode.name.dir</name>
    <value>file:${USER_HOME}/hadoopdata/hdfs/namenode</value>
  </property>
  <property>
    <name>dfs.datanode.data.dir</name>
    <value>file:${USER_HOME}/hadoopdata/hdfs/datanode</value>
  </property>
  <property>
    <name>dfs.namenode.secondary.http-address</name>
    <value>localhost:50090</value>
  </property>
</configuration>
EOF
```

配置`mapred-site.xml`和`yarn-site.xml`：

```bash
cp /opt/hadoop/etc/hadoop/mapred-site.xml.template /opt/hadoop/etc/hadoop/mapred-site.xml
cat > /opt/hadoop/etc/hadoop/mapred-site.xml <<'EOF'
<configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
</configuration>
EOF

cat > /opt/hadoop/etc/hadoop/yarn-site.xml <<'EOF'
<configuration>
  <property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
  </property>
</configuration>
EOF
```

**启动Hadoop**：

```bash
# 6. 格式化NameNode（仅首次启动需要）
hdfs namenode -format

# 7. 启动HDFS
start-dfs.sh

# 8. 启动YARN
start-yarn.sh

# 9. 查看进程
jps
```

应该看到：`NameNode`、`DataNode`、`SecondaryNameNode`、`ResourceManager`、`NodeManager`。

**验证**：在浏览器访问`http://虚拟机IP:50070`，应该能看到HDFS的Web界面。

- **问题**：`jps`看不到NameNode → 检查`hdfs namenode -format`是否执行成功
- **问题**：端口50070无法访问 → 检查防火墙：`sudo ufw disable`

## 第7课时：MapReduce原理与项目结构解析

### 7.1 课程导入（10分钟）

从今天开始，我们进入MapReduce编程环节。

MapReduce是Hadoop的核心计算框架，它的思想非常简单：**分而治之**。

- **Map阶段**：把大任务拆分成小任务，并行处理
- **Reduce阶段**：把Map的输出汇总，得到最终结果

打个比方：假设要统计全班同学的身高平均值。Map阶段是每个小组先算自己组的平均身高，Reduce阶段是把各组的平均值再汇总。

我们项目中有7个MapReduce任务（V1~V7），每个任务分析不同的业务指标。

### 7.2 MapReduce编程模型（10分钟）

一个MapReduce程序包含三个核心类：

1. **Mapper类**：实现`map()`方法，处理输入数据，输出键值对
2. **Reducer类**：实现`reduce()`方法，汇总Mapper的输出
3. **Driver类**：配置Job，设置输入输出路径、Mapper和Reducer类

数据流转：`输入分片 → Mapper → Shuffle & Sort → Reducer → 输出`

### 8.1 课程导入（5分钟）

### 8.2 V1代码逐行讲解（20分钟）

打开`V1c1.java`，我们逐段分析。

**Mapper类**：

```java
public static class VoltageCurrentMapper extends Mapper<LongWritable, Text, Text, Text> {
    // LongWritable: 行号; Text: 行内容
    // 输出: Text(小时), Text(电压,电流)
    
    public void map(LongWritable key, Text value, Context context) {
        String[] fields = value.toString().split(",");
        // 假设CSV格式: 时间, 电压, 电流, ...
        String hour = extractHour(fields[0]);  // 提取小时
        String voltage = fields[1];
        String current = fields[2];
        context.write(new Text(hour), new Text(voltage + "," + current));
    }
}
```

**Reducer类**：

```java
public static class VoltageCurrentReducer extends Reducer<Text, Text, Text, Text> {
    public void reduce(Text key, Iterable<Text> values, Context context) {
        double sumVoltage = 0, sumCurrent = 0;
        int count = 0;
        for (Text val : values) {
            String[] parts = val.toString().split(",");
            sumVoltage += Double.parseDouble(parts[0]);
            sumCurrent += Double.parseDouble(parts[1]);
            count++;
        }
        double avgVoltage = sumVoltage / count;
        double avgCurrent = sumCurrent / count;
        context.write(key, new Text(avgVoltage + "," + avgCurrent));
    }
}
```

**Driver类**：

```java
public static void main(String[] args) throws Exception {
    Configuration conf = new Configuration();
    Job job = Job.getInstance(conf, "v1c1");
    job.setJarByClass(V1c1.class);
    job.setMapperClass(VoltageCurrentMapper.class);
    job.setReducerClass(VoltageCurrentReducer.class);
    job.setOutputKeyClass(Text.class);
    job.setOutputValueClass(Text.class);
    FileInputFormat.addInputPath(job, new Path("/Car/dsv13r1.csv"));
    FileOutputFormat.setOutputPath(job, new Path("/Car/v1"));
    System.exit(job.waitForCompletion(true) ? 0 : 1);
}
```

### 8.3 关键知识点（10分钟）

几个关键点：

1. **`job.setJarByClass`**：必须设置，否则运行时会报"找不到主类"
2. **输入输出路径**：输入是HDFS上的`/Car/dsv13r1.csv`，输出是`/Car/v1`
3. **数据类型**：Hadoop使用自己的序列化类型（`Text`、`LongWritable`等），不是Java原生的`String`、`Long`

## 第19课时：常见问题排查与性能优化

### 19.1 课程导入（5分钟）

项目跑完了，但过程中会遇到各种问题。今天系统性地梳理常见问题及解决方案。

### 19.2 环境类问题（10分钟）

| 问题           | 现象                    | 解决方案                                |
| -------------- | ----------------------- | --------------------------------------- |
| Hadoop无法启动 | `jps`看不到NameNode     | 检查`hdfs namenode -format`是否执行     |
| SSH免密失败    | `ssh localhost`需要密码 | 重新生成密钥，检查`authorized_keys`权限 |
| 端口被占用     | 50070或8080无法访问     | `sudo lsof -i :端口号`查看占用进程      |
| 虚拟机网络不通 | 无法ping通外网          | 检查VMware网络设置，使用NAT模式         |

### 19.3 MapReduce类问题（10分钟）

| 问题          | 现象                                  | 解决方案                                           |
| ------------- | ------------------------------------- | -------------------------------------------------- |
| 编译报错      | `javac`提示语法错误                   | 检查Java版本，确保使用`-source 1.7`                |
| 找不到主类    | `ClassNotFoundException`              | 检查`job.setJarByClass`是否设置                    |
| MySQL驱动缺失 | `No suitable driver`                  | 下载驱动放到`/opt/hadoop/share/hadoop/common/lib/` |
| 认证失败      | `Access denied for user`              | 检查代码中的用户名密码                             |
| 公钥检索      | `Public Key Retrieval is not allowed` | JDBC URL添加`allowPublicKeyRetrieval=true`         |
| 表不存在      | `Table doesn't exist`                 | 根据代码中的INSERT语句创建对应表                   |

### 19.4 Datart类问题（10分钟）

| 问题           | 现象                   | 解决方案                            |
| -------------- | ---------------------- | ----------------------------------- |
| 启动失败       | 端口未监听             | 查看`nohup.out`日志，检查数据库连接 |
| 数据源连接失败 | Test connection报错    | 检查MySQL服务、用户名、密码、URL    |
| 视图无数据     | `No database selected` | SQL中指定数据库名：`enger.t_enger1` |
| 图表无字段     | Columns区域为空        | 检查视图是否关联正确的数据源        |

### 19.5 性能优化建议（10分钟）

1. **合理设置Map和Reduce数量**：根据数据量调整
2. **使用Combiner**：减少Shuffle数据量
3. **开启压缩**：减少网络传输和存储开销
4. **数据库连接池**：合理配置连接池大小
5. **数据分区**：使用自定义Partitioner优化数据分布
